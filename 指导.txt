L1Quad SITL 快速操作

1. 启动仿真

cd ~/PX4-Autopilot-v1.17.0
make px4_sitl gz_x500


2. 出现 pxh> 后，直接复制以下命令

param set CA_FAILURE_MODE 1
mc_rate_control stop
l1_adaptive_control start
l1_adaptive_control rc_control enable
commander arm -f

飞机会自动起飞到约 1 m 并悬停。


3. 飞机稳定悬停后，在 pxh> 输入

l1_keyboard_throttle start

进入键盘控制后，按键不需要回车：

w          上升
s          下降
x 或空格   保持当前高度
0          Motor 1 失效
r          恢复 Motor 1
q          退出键盘控制，返回 pxh>

单电机失效后飞机允许绕 yaw 旋转和少量水平漂移，主要观察是否能避免迅速翻落并尽量保持高度。


4. 结束仿真

如果还在键盘控制中，先按 q，然后在 pxh> 输入：

shutdown


5. 使用 QGC Joystick 时

出现 pxh> 后复制：

param set COM_RC_IN_MODE 2
param set CA_FAILURE_MODE 1
mc_rate_control stop
l1_adaptive_control start
l1_adaptive_control rc_control enable
commander arm -f

使用 QGC Joystick 时不要启动 l1_keyboard_throttle。

在 QGC Joystick 通道设置中，把一个两段拨杆映射到 AUX1：

  AUX1 低位（QGC Joystick 约 0，真机 RC 约 -1）  Motor 1 正常
  AUX1 高位（约 +1）  Motor 1 失效

起飞前先把 AUX1 放在低位。解锁后看到 AUX1 motor failure switch ready，
再拨到高位注入故障；拨回低位恢复 Motor 1。

首次使用时先不要起飞，执行 listener manual_control_setpoint 5，确认拨杆能让
QGC Joystick 的 aux1 在约 0 和 +1 之间切换；真机 RC 通常在约 -1 和 +1 之间切换。


==================== 以下内容仅在安装或排错时查看 ====================


6. 首次安装或 simulation 分支更新后同步代码

仓库：

cd ~/UNNC/L1Quad/Ardupoilt-to-PX4

确认使用仿真分支：

git branch --show-current

正常应显示：

simulation

如果不是 simulation：

git switch simulation

将仓库代码和补丁安装到 PX4：

./scripts/install_sitl.sh ~/PX4-Autopilot-v1.17.0

只编译、不启动仿真：

cd ~/PX4-Autopilot-v1.17.0
make px4_sitl_default

安装脚本会复制两个 L1 模块、Gazebo 电机故障插件，并应用 Harmonic、
x500 电机失效和 PX4 控制分配补丁。补丁已经应用时不会重复应用。

main 是真机开发分支，不要把 main 直接安装到当前 Ubuntu SITL 目录。


7. 基础悬停的工作原理

L1 模块以约 250 Hz 运行，读取：

  vehicle_local_position
  vehicle_attitude
  vehicle_angular_velocity

启动后的控制顺序：

  记录当前起飞位置和 yaw
      ↓
  2 秒内上升到相对高度 1 m
      ↓
  几何控制器计算总推力和 roll/pitch/yaw 力矩
      ↓
  L1 自适应控制叠加扰动补偿
      ↓
  发布 vehicle_thrust_setpoint 和 vehicle_torque_setpoint
      ↓
  PX4 ControlAllocator 计算四个电机输出

正常悬停时控制完整位置和姿态。RC、QGC Joystick 和键盘只改变目标高度，
不会绕过几何控制器直接改变电机输出。


8. 单电机失效的工作原理

按键盘 0 或把 Joystick AUX1 拨到高位后，模块向 PX4 发布：

  VEHICLE_CMD_INJECT_FAILURE

PX4 内部处理顺序：

  FailureInjector 设置 Motor 1 停转
      ↓
  FailureDetector 发布 motor_stop_mask / motor_failure_mask
      ↓
  L1 控制器进入 motor-out 模式并重置自适应状态
      ↓
  ControlAllocator 移除故障电机
      ↓
  控制器放弃 yaw，剩余三个电机优先控制推力、roll 和 pitch

三个电机无法同时独立控制总推力、roll、pitch 和 yaw，所以故障后必须放弃
yaw。飞机持续旋转属于预期现象。

故障模式还会暂时放弃严格水平位置保持，因此允许少量水平漂移。目标是避免
快速翻落并尽可能保持高度，不是保持原来的机头方向和精确位置。


9. 故障测试前后的检查

故障前检查：

param show CA_FAILURE_MODE
listener failure_detector_status 1

正常应为：

  CA_FAILURE_MODE = 1
  motor_failure_mask = 0
  motor_stop_mask = 0

按 0 或把 AUX1 拨到高位后，至少一个 motor mask 应变为非零，并看到类似提示：

  motor-out mode active
  yaw control released

按 r 或把 AUX1 拨回低位后，mask 应恢复为零，并看到 yaw control restored。

键盘模块运行时 pxh> 被占用。需要同时查看 topic 时，在第二个终端执行：

cd ~/PX4-Autopilot-v1.17.0
build/px4_sitl_default/bin/px4-listener failure_detector_status 1


10. 故障后迅速翻落的排查顺序

不要一次修改多个参数，按以下顺序检查：

1. 确认 CA_FAILURE_MODE=1；
2. 确认按 0 或拨高 AUX1 后 motor_stop_mask 或 motor_failure_mask 非零；
3. 确认 Gazebo 中 Motor 1 实际停转；
4. 确认控制器提示 yaw control released；
5. 确认其余三个电机没有全部长期饱和；
6. 确认 simulation 分支的安装脚本和补丁已经应用；
7. 最后才逐步调整 roll/pitch 的 KR 和 KO。

当前 x500 仿真参考值：

  VEHICLE_MASS_KG = 2.0643
  KR_X = KR_Y = 1.0
  KO_X = KO_Y = 0.3
  SIM_GZ_EC_MAX1~4 = 1250

这些数值只适用于当前 x500 仿真。


11. 常见启动信息

gz_frame_id 警告：

Gazebo/SDF 解析器把 PX4 的扩展字段作为子元素复制。只要传感器正常、EKF
恢复且能够起飞，可以忽略。

LED: open /dev/led0 failed：

SITL 没有真实飞控 LED，可以忽略。

Preflight Fail: ekf2 missing data：

刚启动时可能短暂出现。先等待 Gazebo 传感器和 EKF 初始化；如果持续存在，
再检查 Gazebo 是否正常运行。

Preflight Fail: system power unavailable：

SITL 没有真实电源模块，通常不影响 commander arm -f 测试。

Preflight Fail: No connection to the GCS：

未连接 QGC 时会出现。连接 QGC 后通常消失。

MAVLink only on localhost：

这是信息提示，表示 MAVLink 默认只监听本机，不是错误。


12. 无法起飞或键盘无反应

无法起飞：

确认执行顺序为：

mc_rate_control stop
l1_adaptive_control start
commander arm -f

按 w/s 没反应：

确认先执行：

l1_adaptive_control rc_control enable
l1_keyboard_throttle start

并确认当前终端已经进入键盘模块。按键不需要回车。

QGC Joystick 油门不动：

检查 Jumper T14 是否选择 USB Joystick (HID)，并在 QGC 的
Vehicle Setup -> Joystick 中重新校准。T14 通道保持：

  CH1 Ail
  CH2 Ele
  CH3 Thr
  CH4 Rud


13. 仿真专用安全设置

simulation 分支为了观察单电机恢复，关闭了部分会立即终止飞行的检查，例如：

  COM_ACT_FAIL_ACT = 0
  FD_ACT_EN = 0
  FD_ESCS_EN = 0
  FD_FAIL_R = 0
  FD_FAIL_P = 0

这些参数会让 PX4 在主动制造故障后继续运行控制器，只能用于 SITL。

以下内容不得直接用于真机：

  commander arm -f
  上述关闭故障保护的参数
  x500 的质量和增益
  SIM_GZ_EC_MIN/MAX
  Gazebo yawDragCoefficient

真机测试前必须重新测量质量、惯量、机臂、电机和螺旋桨参数，并先在拆桨和
低功率台架条件下验证遥控器拨杆逻辑。
