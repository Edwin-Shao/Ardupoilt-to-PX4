PX4 v1.17.0 L1 控制器 Gazebo UI 仿真指导

用户速览

本流程用于在 Gazebo Classic UI 中测试自定义 PX4 模块：

1. 启动 PX4 + Gazebo UI：

   cd /home/aim/px4_ws/PX4-Autopilot-v1.17.0
   source /home/aim/px4_ws/px4-venv/bin/activate
   unset HEADLESS
   make px4_sitl gazebo-classic_iris

2. 进入 PX4 shell 的 pxh> 后切换到 L1 控制：

   mc_rate_control stop
   l1_adaptive_control start
   commander arm

3. 开启遥控器高度接管：

   l1_adaptive_control rc_control enable
   l1_keyboard_throttle start

4. 键盘控制：

   w  throttle +0.2，目标高度上升
   s  throttle -0.2，目标高度下降
   0  throttle 清零，并关闭 rc_control
   q  退出键盘 throttle 工具

5. 关闭遥控器高度接管：

   如果还在 l1_keyboard_throttle 中，按 0。
   如果已经退出键盘工具，在 pxh> 中执行：

   l1_adaptive_control rc_control disable

6. 终止仿真：

   推荐在 pxh> 中执行：

   shutdown

   也可以在启动仿真的终端中按 Ctrl+C。当前工作区已修复 Ctrl+C 后 PX4 Exiting 卡住的问题，Ctrl+C 会同时清理 PX4、gzserver 和 gzclient。


一、重要说明

1. 本流程只用于 Gazebo 仿真，不要直接用于真实飞机。
2. 当前 L1 控制器是独立 PX4 模块，模块名是 l1_adaptive_control。
3. 当前键盘遥控测试工具是独立 PX4 模块，模块名是 l1_keyboard_throttle。
4. L1 控制器会发布 vehicle_thrust_setpoint 和 vehicle_torque_setpoint。
5. PX4 默认 mc_rate_control 也会发布同类控制量，所以测试 L1 前必须先执行 mc_rate_control stop。
6. 当前 L1 轨迹逻辑是：解锁后自动起飞到约 1 m，然后悬停。
7. 遥控器高度接管只修改目标高度和目标垂直速度，不直接修改推力输出。


二、启动带 UI 的 Gazebo Classic 仿真

打开终端：

cd /home/aim/px4_ws/PX4-Autopilot-v1.17.0
source /home/aim/px4_ws/px4-venv/bin/activate
unset HEADLESS
make px4_sitl gazebo-classic_iris

正常现象：

1. 终端会编译或直接启动 PX4 SITL。
2. Gazebo Classic 图形窗口会打开。
3. Gazebo 中会出现 Iris 四旋翼。
4. 终端最后进入 PX4 shell，提示符为：

pxh>

如果 Gazebo UI 没有出现：

1. 确认执行过 unset HEADLESS。
2. 确认当前环境有图形桌面，DISPLAY 不为空。
3. 如果使用远程桌面，确认远程桌面的图形转发正常。


三、切换到 L1 模式

在 pxh> 中依次执行：

mc_rate_control stop
l1_adaptive_control start
commander arm

预期现象：

1. 飞机解锁。
2. L1 控制器开始发布推力和力矩 setpoint。
3. 飞机自动起飞并悬停在约 1 m。

检查 L1 状态：

l1_adaptive_control status

重点看这些字段：

state_valid_for_control=1
trajectory_valid=1
geometric_output_valid=1
l1_update_executed=1
publish_active=1
publish_count 大于 0

检查位置：

listener vehicle_local_position 1

NED 坐标系中，z 约为 -1.0 表示高度约 1 m。


四、接入遥控器高度控制

在 pxh> 中执行：

l1_adaptive_control rc_control enable
l1_keyboard_throttle start

l1_keyboard_throttle 是前台交互工具。启动后当前 pxh> 会进入键盘读取状态，此时直接按键即可，不需要再输入回车。

按键含义：

w  throttle 增加 0.2，目标高度上升
s  throttle 减少 0.2，目标高度下降
0  throttle 归零，并自动执行 l1_adaptive_control rc_control disable
q  throttle 归零，并退出 l1_keyboard_throttle

注意：

1. throttle 范围限制在 [-1.0, 1.0]。
2. throttle 为 0 时保持当前手动目标高度。
3. throttle 为正时上升，NED target_z 会变小。
4. throttle 为负时下降，NED target_z 会变大。
5. 该工具发布 manual_control_setpoint.throttle，不直接发布推力。

查看遥控输入：

listener manual_control_setpoint 1

查看 L1 是否收到遥控输入：

l1_adaptive_control status

重点看：

rc_height=1
manual=1
valid=1
throttle=当前键盘 throttle


五、去掉遥控器高度控制

方式 1：仍在 l1_keyboard_throttle 中时，直接按：

0

效果：

1. throttle 清零。
2. 自动关闭 rc_control。
3. L1 轨迹重新回到默认 1 m 悬停逻辑。

方式 2：已经退出键盘工具后，在 pxh> 中执行：

l1_adaptive_control rc_control disable

如果只是退出键盘工具但不关闭 rc_control，可以按：

q

此时 throttle 会清零，键盘工具退出；如需让 L1 完全回到默认轨迹控制，还应执行：

l1_adaptive_control rc_control disable

查看当前开关状态：

l1_adaptive_control rc_control status


六、终止仿真

推荐方式：在 pxh> 中执行：

shutdown

这会关闭 PX4 SITL，并触发 Gazebo 清理。

如果终端正在 l1_keyboard_throttle 中：

1. 按 q 退出键盘工具。
2. 在 pxh> 中执行 shutdown。

备用方式：在启动仿真的终端按：

Ctrl+C

当前工作区已修复 Ctrl+C 后卡在 PX4 Exiting 的问题。修复后 Ctrl+C 会触发 sitl_run.sh 的清理逻辑，收掉本次启动的 gzserver 和 gzclient。

如果异常情况下仍有残留进程，可以手动检查：

ps -eo pid,ppid,stat,cmd | rg 'px4|gzserver|gzclient|gazebo'

手动清理：

pkill -x gzclient
pkill -x gzserver
pkill -f build/px4_sitl_default/bin/px4

八、常见问题

问题 1：Gazebo 没有弹出 UI。

处理：

unset HEADLESS
echo $DISPLAY
make px4_sitl gazebo-classic_iris

如果 DISPLAY 为空，需要先进入图形桌面或配置远程桌面。

问题 2：启动 L1 后没有控制输出。

处理：

1. 确认执行了 mc_rate_control stop。
2. 确认执行了 l1_adaptive_control start。
3. 确认执行了 commander arm。
4. 执行 l1_adaptive_control status，检查 publish_active 和 publish_count。

问题 3：按 w/s 没有效果。

处理：

1. 确认执行了 l1_adaptive_control rc_control enable。
2. 确认 l1_keyboard_throttle start 正在运行。
3. 执行 l1_adaptive_control status，检查 rc_height、manual、valid、throttle。
4. 注意 l1_keyboard_throttle 是前台工具，必须让当前终端停在该工具里按键。

问题 4：按 Ctrl+C 后退出。

说明：

Ctrl+C 会终止当前仿真。修复后不应再卡在 PX4 Exiting。若只是想退出键盘工具，不要按 Ctrl+C，按 q。

问题 5：commander disarm 提示 not landed。

说明：

飞机仍在空中时 PX4 会拒绝直接 disarm。仿真测试中建议使用 shutdown 结束。
